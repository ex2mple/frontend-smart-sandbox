# AGENTS.md в песочницах + восемь шаблонов-визуализаторов

**Дата:** 2026-06-23
**Ветка:** `feat/sandbox-mvp`
**Статус:** утверждён дизайн, готов к плану реализации

## Цель

Frontend Smart Sandbox — приложение для быстрой проверки фундаментальных JS/Angular-концептов
с упором на **наглядность** ("мне должно быть наглядно понятно как что-то работает").

Эта работа добавляет две вещи:

1. **AGENTS.md в песочницах** — чтобы агент, запущенный внутри песочницы, сразу был в контексте:
   знал цель проекта, конвенции и конкретный учебный интент данной песочницы.
2. **Восемь шаблонов-визуализаторов** фундаментальных концептов — каждый рабочий и наглядный
   из коробки, как стартовая точка для экспериментов.

Изменений в `tools/sandbox/server.mjs` и `tools/sandbox/codegen.mjs` **не требуется**: новые
шаблоны и файлы AGENTS.md подхватываются автоматически (`listTemplateIds` через `readdir` +
рекурсивный `copyTemplateTree`, который копирует все файлы и прогоняет `renderTemplate`).

## 1. AGENTS.md — два уровня

### Корневой `src/app/sandboxes/AGENTS.md`

Обычный закоммиченный файл (не генерируется). Общий гайд для агента по всему workspace:

- Что это за проект и его цель (быстрая наглядная проверка JS/Angular-концептов).
- Жёсткие конвенции (из `.claude/CLAUDE.md`): standalone-компоненты, `ChangeDetectionStrategy.OnPush`,
  signals + `computed()`, `input()`/`output()` вместо декораторов, native control flow (`@if/@for/@switch`),
  host-bindings в объекте `host` (не `@HostBinding/@HostListener`), `class`/`style`-bindings вместо
  `ngClass/ngStyle`, `inject()` вместо constructor injection, LESS со `var(--sb-*)` токенами,
  a11y (проходит AXE, WCAG AA).
- Контракт движка: каждый шаблон = `<name>.ts/.html/.less/.routes.ts`; маршрут `path: 's/<name>'`;
  **никогда не редактировать `sandbox.routes.generated.ts`** (skip-worktree, переписывается сервером);
  **никогда не `import.meta.glob`** по папкам песочниц.
- Где dev-консоль (`SandboxLog` + оверлей в shell) и как через неё логировать в наглядных демках.
- Где живут шаблоны (`tools/sandbox/templates/`) и как добавить новый (просто папка).

### Per-sandbox `AGENTS.md`

По одному файлу в каждой папке-шаблоне (`tools/sandbox/templates/<tpl>/AGENTS.md`). Копируется
`copyTemplateTree` с подстановкой `{{name}}/{{className}}/{{selector}}/{{title}}`. При "pin"
коммитится вместе с песочницей.

Посадка агента — **«тьютор + инженер»**: объясняет как работает концепт И сразу реализует
эксперименты в скаффолде по конвенциям. Структура:

```md
# {{title}} — sandbox

> Учебная песочница. Ты — и тьютор, и инженер: объясняй как работает <концепт>
> и сразу реализуй эксперименты в этом скаффолде. Общие правила проекта: ../../AGENTS.md

## Что здесь изучается
<1 абзац: суть концепта, ключевая интуиция>

## Что уже в скаффолде
<карта файлов: что где, какие крючки для наглядности уже расставлены>

## Попробуй (эксперименты)
- [ ] <эксперимент 1 — что поменять и что станет видно>
- [ ] <эксперимент 2>
- [ ] <эксперимент 3>

## Ловушки / на что смотреть
<типичные заблуждения по концепту>
```

> Примечание: per-sandbox AGENTS.md лежит в корне песочницы (`generated/<name>/AGENTS.md`),
> а корневой — в `src/app/sandboxes/AGENTS.md`. Относительный путь между ними всегда
> `../../AGENTS.md` (для всех шаблонов, включая multipage — файл AGENTS.md один на песочницу,
> в её корне, не в подпапках `pages/`).

Три существующих шаблона (`blank`, `example`, `multipage`) тоже получают тонкий generic
per-sandbox AGENTS.md.

## 2. Планка качества и общие контракты шаблонов

Каждый шаблон-визуализатор:

- **Рабочий и наглядный из коробки** — открыл `/s/<name>`, сразу видно поведение; минимально-но-настоящий,
  приглашает дорабатывать (не законченный продукт).
- Все 4 файла (`__name__.ts/.html/.less/.routes.ts`) + `AGENTS.md`.
- Логики в шаблоне минимум; стили только через `var(--sb-*)` токены.
- Все конвенции CLAUDE.md (OnPush, signals, native control flow, host-объект, a11y).
- Где уместно — пишет в dev-консоль через `SandboxLog`, чтобы поведение читалось и в логе.

## 3. Восемь шаблонов

### Angular-внутренности

- **change-detection** — дерево из 3–4 вложенных компонентов, у каждого видимый счётчик
  ре-рендеров (CD-tick, инкремент в шаблоне); кнопки запускают CD через событие / `setTimeout` /
  set сигнала; тумблер `OnPush ↔ Default` на поддереве. Видно *кто* перерисовался и почему.
- **signals** — `signal`-входы → цепочка `computed` → `effect`; панель «когда что пересчиталось /
  effect побежал»; демо glitch-free и `untracked`.
- **di-tree** — сервис с instance-id; провайдеры на уровнях root / component / element; каждый
  уровень печатает, какой инстанс ему достался; кнопки-демо `@Optional` / `@SkipSelf` / `@Self`.
- **lifecycle** — child с тумблером mount/unmount и сменой `input`; таймлайн-лог порядка хуков
  (`OnChanges → OnInit → DoCheck → AfterContentInit → AfterViewInit → … → OnDestroy`).

### JS-фундамент

- **event-loop** — кнопки добавляют sync / `Promise.then` / `queueMicrotask` / `setTimeout`;
  пошаговый прогон с тремя колонками (call stack / microtasks / macrotasks) и итоговым порядком вывода.
- **this-binding** — одна функция, вызванная как метод / отдельно / через `bind` / стрелка;
  рядом печатается, чему равен `this`.
- **prototype-chain** — объект и его цепочка `[[Prototype]]`; наглядный проход вверх по цепи при
  доступе к свойству/методу + property shadowing.
- **closures** — фабрики-счётчики, захват переменной в цикле (`let` vs `var`), module-pattern;
  видно изоляцию состояния.

## 4. План сборки

- **Фан-аут субагентами (Sonnet):** 8 агентов, одна папка шаблона на агента (disjoint-зоны,
  без git-коммитов агентами). Каждому — самодостаточный промпт с контрактами: имена `--sb-*`
  токенов, структура 4 файлов, формат routes (`path: 's/<name>'`, `__name__` в путях,
  `{{name}}/{{className}}/{{selector}}/{{title}}` в содержимом), посадка AGENTS.md.
- **Параллельно** оркестратор пишет корневой `src/app/sandboxes/AGENTS.md` + generic
  per-sandbox AGENTS.md для `blank`/`example`/`multipage`.
- **Гейты (оркестратор):** `npm run test:codegen` (vitest) + `ng build` + живой e2e — создать
  по одной песочнице из каждого нового шаблона, проверить, что `/s/<name>` грузится и AGENTS.md
  на месте с подставленными значениями. Коммит по зонам.
- Изменений в `server.mjs` / `codegen.mjs` не требуется.

## Риски / открытые вопросы

- 8 шаблонов — большой батч. Митигируется фан-аутом и единым контрактом. Планка качества —
  «минимально-но-настоящий», не законченный продукт.
- `create` пока не чистит частично записанную папку при ошибке (известный гэп, вне scope).
