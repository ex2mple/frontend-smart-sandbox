Ловушка: мутация без signal

Кнопка меняет обычное поле класса (не signal). Angular ничего об этом не узнаёт — ни один CD-проход не планируется. Значение изменится «под капотом» незаметно; на экране оно обновится только тогда, когда какой-то ДРУГОЙ триггер выше вызовет реальный проход.

Default ChangeDetectionStrategy.Default

Перепроверяется на каждом CD-проходе, дошедшем до него, независимо от изменения inputs.

Как это работает

  • checked N times — сколько раз Angular реально выполнил шаблон карточки за сессию (dirty-check). Считает родитель внутри окна одного CD-прохода и присылает готовое число карточке через input — сама карточка счётчик не ведёт.
  • Клик снаружи — обновляет родительский signal вне карточек; проход доходит до Default-узлов (проверяются всегда) и до OnPush-узлов (их inputs не менялись — пропускаются).
  • Изменить sharedInput — меняет input(), переданный в обе группы карточек; OnPush-узлы теперь тоже проверяются, потому что изменился их input.
  • setTimeout → signal — приложение zoneless, сам таймер CD не запускает. Через 500 мс колбэк пишет в signal — именно эта запись планирует проход.
  • Клик внутри карточки — кнопка ВНУТРИ OnPush-узла пишет в его собственный signal → компонент сам помечает себя dirty. Соседняя OnPush-карточка при этом не меняется и остаётся skipped.
