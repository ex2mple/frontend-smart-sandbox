Каждая карточка показывает, сколько раз Angular реально выполнил её шаблон (dirty-check). При каждой проверке карточка вспыхивает.

parent counter: {{ parentCounter() }} shared input: {{ sharedInput() }}

Default ChangeDetectionStrategy.Default

Перепроверяется на каждом CD-проходе, дошедшем до него, независимо от изменения inputs.

@for (node of defaultNodes(); track node.label) { }

OnPush ChangeDetectionStrategy.OnPush

Перепроверяется только когда изменился input или внутри компонента сработал signal / событие.

@for (node of onpushNodes(); track node.label) { }

Как читать счётчики

  • checked N times — сколько раз Angular реально выполнил шаблон компонента (dirty-check). Счётчик пишется в signal после прохода (через микротаск), и перерисовки от самого счётчика не учитываются.
  • Local event — обновляет родительский signal; CD-проход доходит до родителя, Default-узлы перепроверяются, а OnPush-узлы пропускаются: их inputs не изменились.
  • setTimeout → signal.set — приложение zoneless, поэтому сам по себе setTimeout CD не запускает. Через 500 мс колбэк пишет в signal — именно эта запись планирует CD-проход. Если бы колбэк менял обычное поле, ничего бы не перерисовалось.
  • Set signal input — меняет sharedInput, переданный как input() в оба поддерева; OnPush-узлы перепроверяются, потому что изменился их input.
