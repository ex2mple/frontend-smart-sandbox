## Что здесь изучается

Angular запускает **Change Detection (CD)** — обход дерева компонентов — при каждом событии, таймере или завершении промиса. При `ChangeDetectionStrategy.Default` Angular проверяет компонент на каждом проходе; при `OnPush` — только когда изменился ссылочный input, сработал signal внутри компонента или явно вызван `markForCheck()`. Счётчик «checked N times» на каждой карточке отображает это буквально: он растёт только тогда, когда Angular действительно заходит в шаблон компонента.

## Что уже в скаффолде

| Файл | Содержимое |
|------|-----------|
| `__name__.ts` | Главный компонент `{{className}}` (OnPush) + два дочерних класса в том же файле: `CdNodeDefault` (Default) и `CdNodeOnPush` (OnPush). |
| `__name__.html` | Два поддерева (`@for` по `defaultNodes` / `onpushNodes`), три кнопки-триггера, живые счётчики parent counter / shared input, легенда. |
| `__name__.less` | Flash-анимация карточки при каждой проверке (`.cd-node.flash`), бейджи стратегий. |
| `__name__.routes.ts` | Ленивый маршрут `path: ''`. |

**Ключевые крючки видимости:**
- `renderStamp()` — обычный метод (не signal), вызывается прямо в шаблоне; Angular вызывает его при каждой грязной проверке компонента, тем самым инкрементируя счётчик.
- `[class.flash]="flashing"` + `setTimeout(180 ms)` — визуальная вспышка карточки при каждой проверке.
- `console.log` в каждом триггере — видно в оверлее dev-консоли.

## Ловушки / на что смотреть

- **`renderStamp()` — это не сигнал**. Его инкремент не триггерит CD сам по себе, он лишь фиксирует факт, что Angular уже зашёл в шаблон. Не путай со сигналом, который сам является триггером.
- **OnPush ≠ «никогда не проверяется»**. OnPush только пропускает компонент, если Angular не видит «грязного» маркера. При изменении `input()` Angular автоматически помечает компонент, поэтому OnPush-узлы всё равно перепроверяются при «Set signal input».
- **setTimeout без zone.js**: в `zoneless`-приложении `setTimeout` не запускает CD автоматически; нужен `markForCheck` или изменение сигнала. В этом скаффолде zone.js активна, поэтому `setTimeout` работает как ожидается.
- **`flashing = true` внутри `renderStamp()`**: это мутация обычного поля, а не сигнала — Angular не видит её как триггер. Флаг подхватывается только потому, что Angular уже находится в текущем цикле проверки. В компонентах с OnPush это работает корректно только при уже запущенном CD-проходе.
