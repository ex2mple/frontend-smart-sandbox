## Что здесь изучается

Angular запускает **Change Detection (CD)** — обход дерева компонентов. Приложение **zoneless** (без zone.js): CD планируется только когда меняется signal, срабатывает обработчик события из шаблона или вызван `markForCheck()`. Таймеры и промисы **сами по себе** CD не запускают. При `ChangeDetectionStrategy.Default` Angular проверяет компонент на каждом проходе, дошедшем до него; при `OnPush` — только когда изменился input, сработал signal внутри компонента или явно вызван `markForCheck()`. Счётчик «checked N times» на каждой карточке отображает это буквально: он растёт только тогда, когда Angular действительно выполняет шаблон компонента.

## Что уже в скаффолде

| Файл | Содержимое |
|------|-----------|
| `__name__.ts` | Главный компонент `{{className}}` (OnPush) + два дочерних класса в том же файле: `CdNodeDefault` (Default) и `CdNodeOnPush` (OnPush). |
| `__name__.html` | Два поддерева (`@for` по `defaultNodes` / `onpushNodes`), три кнопки-триггера, живые счётчики parent counter / shared input, легенда. |
| `__name__.less` | Flash-подсветка карточки (`.cd-node.flash`, класс на 300 мс от сигнала `flashing`), бейджи стратегий. |
| `__name__.routes.ts` | Ленивый маршрут `path: ''`. |

**Ключевые крючки видимости:**
- `recordCheck()` — вызывается прямо из шаблона, то есть ровно тогда, когда Angular реально выполняет шаблон компонента. Он **не пишет в signal во время проверки** (в zoneless это дало бы бесконечный цикл и NG0100): проверки копятся в обычном поле `pendingChecks` и переносятся в signal `checkCount` одним `queueMicrotask`-флашем после прохода.
- `selfInflicted` — перерисовки, вызванные самим флашем и снятием вспышки, помечаются и **не считаются**, иначе счётчик раскручивал бы сам себя (запись в `checkCount` → новая проверка → новая запись → …).
- `[class.flash]="flashing()"` — signal, ставится в `true` при флаше и в `false` через `setTimeout(300 ms)`; вспышка = реальная проверка, зафиксированная счётчиком.
- `console.log` в каждом триггере — видно в оверлее dev-консоли.

## Ловушки / на что смотреть

- **`recordCheck()` не пишет в signal синхронно**. Запись в signal во время выполнения шаблона запланировала бы новый CD-проход прямо из текущего (бесконечный цикл) и рискует NG0100. Поэтому — буфер в обычном поле + один `queueMicrotask`-флаш после прохода. Это обязательный паттерн инструментирования в zoneless-приложении.
- **Наблюдение возмущает систему**: сам флаш счётчика — тоже запись в signal, которая перерисовывает карточку. Такие «свои» проходы помечены `selfInflicted` и не считаются — иначе «checked N times» рос бы сам по себе.
- **OnPush ≠ «никогда не проверяется»**. OnPush только пропускает компонент, если Angular не видит «грязного» маркера. При изменении `input()` Angular автоматически помечает компонент, поэтому OnPush-узлы всё равно перепроверяются при «Set signal input».
- **setTimeout в zoneless не запускает CD**. Приложение работает без zone.js, поэтому кнопка «setTimeout → signal.set» демонстрирует не «магию таймера», а отложенную запись в signal: CD планирует именно `parentCounter.update()` внутри колбэка. Замени запись в signal на мутацию обычного поля — и ничего не перерисуется.
- **Хуки `ngDoCheck`/`ngAfterViewChecked` не годятся как счётчик проверок**: они срабатывают у ребёнка каждый раз, когда проверяется его родитель, — даже если OnPush-ребёнок был пропущен. Честный счётчик «шаблон реально выполнен» — только вызов из самого шаблона.
