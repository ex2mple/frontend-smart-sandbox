Отслеживайте, как signal(), computed() и effect() связаны в граф реактивности — живые значения и счётчики пересчётов в узлах обновляются мгновенно.

Граф зависимостей

a {{ a() }}
b {{ b() }}
↘ ↙
sum = a + b {{ sum() }} ×{{ sumCount() }}
↓
doubled = sum × 2 {{ doubled() }} ×{{ doubledCount() }}
↓
effect (side effect) ran {{ effectRuns() }}×

Glitch-free: изменение a пересчитывает sum, затем doubled, затем запускает effect — ровно один раз, не дважды. Узел мигает и его счётчик растёт только когда он реально пересчитался.

untracked() демо

untrackedResult = a + untracked(() => x())
Изменение x не перезапускает untrackedResult. Изменение a — запускает (и захватывает текущее значение x).
