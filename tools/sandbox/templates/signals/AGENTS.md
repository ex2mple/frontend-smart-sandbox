# {{title}} — sandbox

> Учебная песочница. Ты — и тьютор, и инженер: объясняй как работают signals (signal, computed, effect, untracked), когда и почему происходит пересчёт, что такое «glitch-free» реактивность.
> Общие правила проекта: ../../AGENTS.md

## Что здесь изучается

- **`signal()`** — атомарная реактивная ячейка; при `set()` / `update()` с тем же значением пересчёт не происходит.
- **`computed()`** — ленивый кэшированный вычислитель; автоматически отслеживает зависимости и пересчитывается ровно один раз после батча изменений («glitch-free»).
- **`effect()`** — сайд-эффект, подписывается на всё, что читает; работает в injection context (поле компонента).
- **`untracked()`** — чтение сигнала без регистрации зависимости, изменения этого сигнала не перезапускают `computed`/`effect`.
- **Счётчики пересчётов по узлам** — сколько раз РЕАЛЬНО пересчитался каждый `computed` и сколько раз реально сработал `effect`.
- **Глитч-фри демо** — клик по «Обновить A и B» пишет оба базовых сигнала в одном обработчике и доказывает, что `sum`/`doubled` пересчитываются один раз, а не дважды.
- Визуальный граф зависимостей: `a`, `b` → `sum` → `doubled` → `effect`.

## Как устроена демка (архитектура)

- **Живой граф.** `a`, `b`, `sum`, `doubled`, `effect` — обычные сигналы/computed/effect компонента, ничего не подделывается. Узел графа мигает и его счётчик растёт только когда он РЕАЛЬНО пересчитался: `sum`/`doubled` растят обычную (не-сигнальную) переменную `sumRecomputes`/`doubledRecomputes` прямо внутри своего `computed`-тела, `effect` растит сигнал `effectRuns` изнутри своего тела (писать сигнал из `effect` можно — в отличие от `computed`, это поддерживаемый Angular паттерн). Отображение (flash + счётчик в UI) отложено через `queueMicrotask`, потому что приложение zoneless: писать сигнал СИНХРОННО внутри тела `computed` запрещено движком.
- **Реальный эксперимент, записанный.** Кнопка «Обновить A и B» (`../../shared/learning`, `RunRecorder<GraphSnapshot>`) выполняет НАСТОЯЩИЙ клик-сценарий: `a.set()` и `b.set()` в одном синхронном обработчике, затем — СНАРУЖИ реактивного контекста (`queueMicrotask`, затем `setTimeout`) — реальные чтения `sum()`/`doubled()`/`effectRuns()`, которые форсируют пересчёт и позволяют честно продиффить счётчики до/после. Каждый шаг — снимок `GraphSnapshot` (`{ a, b, sum, doubled, sumRecomputes, doubledRecomputes, effectRuns, activeNode }`), иммутабельный (`Object.freeze`).
- **Степпер-реплей.** После завершения клика (эффект точно отработал) шаги копируются ОДИН раз в `replaySteps`, и `<sb-stepper>` проигрывает запись: Шаг / Авто / Сначала. Небольшая панель-снимок над степпером подсвечивает узел, который «сработал» на текущем шаге (`activeNode`); позиция −1 показывает состояние прямо перед кликом.
- **Карточка-предсказание.** `<sb-experiment-card>` спрашивает «сколько раз пересчитается sum» до клика; `actualIndex` вычисляется из РЕАЛЬНО посчитанной разницы счётчика `sumRecomputes` (а не захардкожен) — в норме это ровно 1, но карточка честно покажет 0/2/«зависит», если реальный прогон вдруг даст другой результат.
- **`runId`-токен**: клик по «Обновить A и B» инкрементирует `runId`; отложенные колбэки устаревшего клика (после Reset или повторного клика) становятся no-op.

## Что уже в скаффолде

- `a = signal(1)`, `b = signal(2)` — два базовых сигнала.
- `sum = computed(() => a() + b())` — вычисляемая сумма, растит `sumRecomputes`.
- `doubled = computed(() => sum() * 2)` — вторичный вычислитель, растит `doubledRecomputes`.
- `effect` читает `doubled`, логирует в консоль и увеличивает счётчик `effectRuns`.
- Кнопка «a.set(a())» — демонстрирует отсутствие пересчёта при неизменном значении.
- `x = signal(10)` + `untrackedResult = computed(() => a() + untracked(() => x()))` — изменение `x` не вызывает пересчёт.
- «Обновить A и B» — записанный эксперимент с карточкой-предсказанием и степпером-реплеем (см. выше).

## Попробуй (эксперименты)

- [ ] Нажимай «+» рядом с `a` несколько раз и смотри, что счётчик `effect runs` растёт ровно на единицу за каждое нажатие, а счётчики `sum`/`doubled` в графе — тоже.
- [ ] Нажми «a.set(a())» — убедись, что ни один узел графа не мигнул и ни один счётчик не вырос: Angular видит, что значение не изменилось, и не пересчитывает ничего.
- [ ] В карточке «сколько раз пересчитается sum» выбери вариант, нажми «Обновить A и B» — проверь себя по записи. Реальный ответ: 1 (даже при двух `set()` в одном клике).
- [ ] Прощёлкай степпером весь реплей клика: `set a` и `set b` идут БЕЗ пересчёта sum/doubled (снимок-панель это подтверждает — счётчики те же, что «до»), и только следующий шаг `recompute sum` реально их меняет.
- [ ] Нажми «Обновить A и B» несколько раз подряд — каждый раз ровно 1 пересчёт `sum`, несмотря на то, что оба сигнала меняются вместе.
- [ ] Нажми «x++» несколько раз, затем измени `a` — `untrackedResult` «прыгнет» сразу на сумму всех пропущенных изменений `x`.
- [ ] Добавь третий базовый сигнал `c` и включи его в `sum`; проверь, что изменение `c` тоже запускает `effect` ровно один раз.

## Ловушки / на что смотреть

- **Сайд-эффекты внутри `computed`** — в этом скаффолде `console.info` и отложенный `pulse()` внутри computed добавлены только для обучения. В реальном коде computed-фабрика обязана быть чистой: никаких сайд-эффектов, мутаций, запросов.
- **Писать сигнал внутри `computed` нельзя вообще** — Angular бросит ошибку. Поэтому счётчики пересчётов `sum`/`doubled` — это обычные (не-сигнальные) поля класса; в сигнал для UI они копируются отложенно, вне тела computed.
- **Zoneless: не пиши в сигналы синхронно во время вычисления** — прямой `set()` во время change detection ломает реактивный цикл. Запись шагов эксперимента (`RunRecorder`) вызывается из обработчика клика и из `queueMicrotask`/`setTimeout` колбэков — никогда изнутри `computed`/`effect`.
- **`computed` ленивый** — `sum`/`doubled` не пересчитываются в момент `set()`; пересчёт откладывается до следующего реального чтения. Именно поэтому два `set()` подряд дают один пересчёт, а не два — это и есть glitch-free.
- **`effect` нужен injection context** — создавай эффекты в поле класса или в `constructor`/`ngOnInit` (через `inject`), но не в методах-обработчиках.
- **Бесконечный цикл** — `effect(() => { mySignal.set(...) })` без `untracked` приведёт к ошибке Angular. Всегда оборачивай обратную запись в `untracked`.
- **Равенство при `set`** — Angular сравнивает примитивы по значению; для объектов/массивов потребуется кастомный `equal`-компаратор, иначе пересчёт произойдёт всегда.
