## Что здесь изучается

- **Иерархия инжекторов Angular**: как токен разрешается вверх по дереву компонентов.
- **`providedIn: 'root'`** — синглтон на уровне приложения; все компоненты без локального провайдера получают один и тот же экземпляр.
- **`providers: [SomeService]` на компоненте** — создаёт _новый_ экземпляр для этого компонента и всех его потомков (поддерево делит один экземпляр).
- **`@Self`** (`inject(T, { self: true })`) — искать только в инжекторе _текущего_ узла; ошибка, если провайдера нет.
- **`@SkipSelf`** (`inject(T, { skipSelf: true })`) — пропустить _только_ инжектор текущего узла и начать обычный поиск вверх с родителя. Это не «пропустить ближайшего предка»: выигрывает первый провайдер, найденный у родителей.
- **`@Optional`** (`inject(T, { optional: true })`) — не бросать ошибку, если токен не найден; вернуть `null`.
- **Уникальный идентификатор экземпляра** — счётчик `seq` в модуле показывает, когда сервис создаётся повторно.

## Что уже в скаффолде

| Элемент | Описание |
|---|---|
| `TokenService` | Сервис с `providedIn: 'root'` и уникальным `id` (`++seq`). |
| `ExtraService` | Сервис _без_ провайдера (нигде не зарегистрирован). |
| Root ({{className}}) | Инжектирует root-синглтон `TokenService` (id = #1). |
| `DiBranch` | Объявляет `providers: [TokenService]` — получает новый экземпляр (#2); демонстрирует `@Self` и `@SkipSelf`. |
| `DiGrandchild` | Нет своего провайдера — наследует экземпляр Branch (#2); демонстрирует ловушку `@SkipSelf`: пропускается только его собственный (пустой) инжектор, поэтому skipSelf тоже возвращает Branch (#2), а не Root. |
| `DiOptionalNode` | Инжектирует `ExtraService` с `{ optional: true }` → `null`. |
| Карточки с `di-facts` | Каждый узел показывает id экземпляра и пояснение. |
| `console.info` в `ngOnInit` | В dev overlay видно, какой экземпляр получил каждый компонент. |

## Попробуй (эксперименты)

- [ ] Убери `providers: [TokenService]` из `DiBranch` — проверь, что Branch и Grandchild теперь тоже получают Root-синглтон (#1).
- [ ] Добавь `providers: [TokenService]` на `DiGrandchild` — убедись, что обычный `inject` теперь создаёт _третий_ экземпляр (#3), а `skipSelf` по-прежнему возвращает Branch (#2): теперь пропускается уже непустой собственный инжектор, и результат наконец отличается от обычного `inject`.
- [ ] Добавь `ExtraService` в `providers` компонента `DiOptionalNode` — убедись, что `@Optional` больше не возвращает `null`, а создаёт новый экземпляр.
- [ ] Попробуй `inject(TokenService, { self: true })` в компоненте _без_ локального провайдера — посмотри на ошибку `NullInjectorError` в консоли.

## Ловушки / на что смотреть

- **`@Self` без провайдера = ошибка**: Angular бросит `NullInjectorError` во время инициализации компонента. Используй совместно с `{ optional: true }` если провайдер необязателен.
- **`@SkipSelf` ≠ «пропустить ближайшего предка»**: пропускается только инжектор _текущего_ узла, дальше идёт обычный поиск вверх. Grandchild с `skipSelf` получает экземпляр Branch (#2), а не Root — его собственный инжектор пуст, и пропуск ничего не меняет.
- **Счётчик `seq` — модульный, а не сервисный**: он увеличивается в инициализаторе поля (`readonly id = ++seq`), а не в конструкторе; при горячей перезагрузке модуля (HMR) нумерация начнётся заново.
- **`providers` — не `viewProviders`**: `providers` доступны как в шаблоне компонента, так и в content projection. `viewProviders` — только в шаблоне компонента (не доступны projected контенту).
- **`@SkipSelf` в root** — если вызвать `inject(T, { skipSelf: true })` в компоненте уровня root (нет родителя с нужным токеном), получишь ошибку или `null` при `{ optional: true }`.
- **Изменение `providers` во время выполнения**: массив `providers` фиксируется при компиляции; нельзя динамически добавить или удалить провайдер для уже созданного компонента.
