# {{title}} — sandbox

> Учебная песочница. Ты — и тьютор, и инженер: объясняй, как Angular резолвит `inject()` вверх по дереву инжекторов, что делают `providedIn: 'root'` и `providers: [...]` на компоненте, и как модификаторы `@Self` / `@SkipSelf` / `@Optional` меняют это разрешение. Отвечай на вопросы, помогай с экспериментами и указывай на типичные ловушки.
>
> Общие правила проекта: ../../AGENTS.md

## Что здесь изучается

- **Иерархия инжекторов Angular**: `Root → Branch → Grandchild`; токен резолвится вверх, пока не найдётся провайдер.
- **`providedIn: 'root'`** — `TokenService` создаётся один раз на всё приложение (Root-синглтон).
- **`providers: [TokenService]` на `DiBranch`** — новый экземпляр для этого узла и всего его поддерева (Branch и Grandchild делят один экземпляр, если Grandchild не переопределяет провайдер).
- **Модификаторы запроса** у Grandchild: обычный `inject()`, `{ self: true }`, `{ skipSelf: true }`, `{ optional: true }` (на `ExtraService`, который нигде не зарегистрирован).
- **Ключевая ловушка**: `@SkipSelf` пропускает ТОЛЬКО собственный инжектор узла, где он вызван — резолюция продолжается с ближайшего родителя, а не «прыгает через голову» сразу к деду.

## Как устроена демка (архитектура)

- **Реальный обход, записанный.** Все ветки — обычный `inject()`, `inject(T, { self: true })`, `inject(T, { skipSelf: true })`, `inject(ExtraService, { optional: true })` — реально выполняются в конструкторах/полях `Root` / `DiBranch` / `DiGrandchild` при построении дерева компонентов. Их результаты (id экземпляров, текст `NullInjectorError`, `null`) складываются в `InstanceReportBus` — служебную шину-репортёр (`providedIn: 'root'`), которая НЕ участвует в демонстрируемой иерархии и не влияет на резолюцию.
- **Обход строится из настоящих данных.** По клику «Запустить обход» `buildHops()` собирает шаги для выбранного модификатора, используя реально записанные в `bus` id — топология (кто у кого родитель) зашита в коде, но какой узел «найден» и какой id показан, берётся из настоящих значений `inject()`, а не хардкодится.
- **Степпер-реплей.** Шаги пишутся в `RunRecorder`, после `queueMicrotask` копируются один раз в `replaySteps`, и `<sb-stepper>` проигрывает их (Шаг / Авто / Сначала) поверх дерева: узел получает статус `passed-empty` (проверен, пусто), `skipped` (не проверялся — пропущен модификатором), `found` или `not-found`, плюс pulse-подсветку активного на этом шаге узла. У каждого статуса есть текстовая подпись (не только цвет — WCAG).
- **Карточка-предсказание.** `<sb-experiment-card>` задаёт вопрос в зависимости от выбранного модификатора; `actualIndex` вычисляется из последнего реально записанного шага обхода (`resolveCardAnswer`), а не хардкодится.
- **Branch всегда показывает свою пару фактов** (`inject(TokenService)` и `{ skipSelf: true }`) независимо от того, какой модификатор выбран для Grandchild — это отдельная демонстрация «SkipSelf на непустом инжекторе»: у Branch свой провайдер ЕСТЬ, поэтому `skipSelf` пропускает именно его и находит Root. Контраст с Grandchild, где свой инжектор и так пуст.
- Переключение модификатора кнопками сбрасывает текущую запись и переоткрывает карточку предсказания (`reopenPrediction`); **Reset** возвращает модификатор к «по умолчанию» и тоже сбрасывает всё.

## Попробуй (эксперименты)

- [ ] Оставь модификатор «по умолчанию», предскажи ответ в карточке, нажми **Запустить обход** — проверь: Grandchild помечается «пропущен (пусто)», Branch — «найден», в баннере результата — id экземпляра Branch.
- [ ] Переключись на **@SkipSelf** и предскажи заново — обрати внимание, что правильный ответ НЕ меняется (снова Branch). В объяснении карточки прочитай, почему: у Grandchild `skipSelf` пропускает только его собственный (и без того пустой) инжектор, поэтому поиск наверх идёт точно так же, как без модификатора, и находит Branch — а не сразу Root.
- [ ] Переключись на **@Self** — запусти обход и посмотри на настоящую `NullInjectorError` в карточке узла Grandchild (родителей `@Self` не проверяет вовсе, своего провайдера нет).
- [ ] Переключись на **@Optional (ExtraService)** — прощёлкай степпером все три шага: Grandchild → Branch → Root, нигде провайдера нет → на Root обход заканчивается, результат `null`.
- [ ] Сравни факты `{ skipSelf: true }` у Grandchild и у Branch: у Grandchild результат совпадает с обычным `inject` (Branch), у Branch — меняется на Root, потому что там пропускается непустой собственный провайдер.
- [ ] Прощёлкай степпером (Шаг / Авто / Сначала) весь обход и следи, как pulse-подсветка перемещается по дереву, а статус-чип на узле объясняет, что произошло именно на этом шаге.
- [ ] Нажми **Reset** — модификатор вернётся к «по умолчанию», карточка и степпер очистятся.

## Ловушки / на что смотреть

- **`@SkipSelf` ≠ «пропустить ближайшего предка»**: пропускается ТОЛЬКО инжектор узла, где вызван `inject()`, дальше — обычный поиск вверх, побеждает первый найденный провайдер. У Grandchild (свой инжектор пуст) `skipSelf` всё равно даёт Branch; эффект отличается только у Branch, где свой провайдер есть и именно он пропускается — тогда результат меняется на Root.
- **`@Self` без провайдера = реальная ошибка**: демо не подделывает `NullInjectorError` — она ловится `try/catch` прямо в поле компонента (контекст внедрения там ещё активен), чтобы не уронить страницу, но текст ошибки настоящий.
- **`@Optional` не значит «сразу null»**: `ExtraService` не зарегистрирован ни у одного узла, поэтому обход реально доходит до конца дерева (Root), и только там `{ optional: true }` возвращает `null` вместо `NullInjectorError`.
- **Снимок шага — это состояние ПОСЛЕ него**: активный (pulse) узел — тот, что разбирается на этом шаге; статус «найден» / «не найден» уже проставлен именно на этом шаге, а не на следующем.
- **Zoneless**: `replaySteps` заполняется один раз через `queueMicrotask` после того, как `RunRecorder` сам сбросил запись — не пишите в сигналы синхронно во время построения дерева/CD.
